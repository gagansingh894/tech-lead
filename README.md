# Tech Lead

A personal knowledge base of interactive visual explainers for engineering concepts. Built with Next.js 15, TypeScript, and Tailwind CSS.

## Purpose

These outputs serve as an **authoritative technical reference** — usable across four contexts:

- **Learning** — building deep, interview-ready understanding of each concept
- **Applying** — making architecture decisions on the job with concrete tradeoffs
- **Teaching** — onboarding juniors, running tech talks, writing ADRs
- **Sharing** — artifacts clean enough to drop into team wikis, Notion, or Slack

The standard is tech lead, not tutorial. Every topic covers architecture diagrams, authoritative definitions, polyglot implementations, failure modes, and the questions you ask in design review.

## What it is

Each topic is a self-contained interactive React component — diagrams, code tabs, concept cards, and leadership angles — generated by an AI agent and rendered at its own route. The home page acts as a navigation hub across all topics.

## Getting Started

```bash
# Install dependencies
npm install

# Start dev server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Adding a New Topic

1. Drop a new TSX component into `src/topics/`
2. Add `"use client"` as the first line (components use React hooks)
3. Add one entry to `src/lib/registry.ts`:

```ts
{
  slug: 'your-topic-slug',
  title: 'Your Topic Title',
  description: 'One sentence description.',
  tag: 'Category',
  component: YourComponent,
}
```

The home page tile and route are automatically created — no other changes needed.

## Topic Component Contract

Each topic component is fully self-contained:

- Owns its own layout, styles, and state
- Uses inline styles (not Tailwind) for full visual control
- Must start with `"use client"` if it uses any React hooks
- No props required — renders standalone
- The shared `BackButton` is injected by the route wrapper, not the component itself
